# 飞书文档评论入口（/subscribe-lark-doc）

把一篇**飞书云文档**变成会话的输入 / 输出通道：订阅文档后，文档里的**评论**会作为消息喂进会话，机器人的回复直接发表回**该评论的讨论串**里。

不用离开正在写的文档，在评论里 `@机器人` 提个问题、让它改点东西，回复就出现在评论区——适合「边看文档边让 AI 干活」「在文档里就地协作」。

## 怎么用

1. 在飞书话题里**正常发起一个会话**（入口不变，状态卡 / 终端链接等都在这里）。
2. 会话内发命令订阅文档：

   ```
   /subscribe-lark-doc <飞书文档链接>
   ```

   首次会引导你完成「文档权限」授权（见下「授权」）。
3. 之后在那篇文档里**评论并 @机器人**，评论就会作为消息喂进这个会话；机器人的回复会**发表回该评论的讨论串**（以机器人身份，并 `@` 回你）。

| 命令 | 说明 |
|------|------|
| `/subscribe-lark-doc <文档链接>` | 订阅该文档，把它的评论接进当前会话 |
| `/subscribe-lark-doc list` | 查看当前会话已订阅的文档 |
| `/subscribe-lark-doc off` | 退订当前会话的所有文档 |

> 支持飞书云文档（docx）与知识库 wiki 链接。

## 交互形式

- **输入**：文档评论（默认需 `@机器人` 才触发）→ 当作消息喂进绑定的会话，和在群里发消息等价。
- **输出（回复）**：机器人对「来自文档评论的那一轮」的回复，**发回到该评论的讨论串**里——
  - 以**机器人身份**发表（不是你的身份）；
  - 默认 `@` 回评论发起人；
  - 超长回复自动分段成多条。
- **状态卡 / 终端链接卡 / 按钮**：仍发在**会话起点**（飞书话题），不进文档评论（文档评论是纯文本，承载不了富卡片）。

一条会话可订阅多个文档；一个文档同时只绑一条活跃会话。

## 触发范围（per-bot，可在 Dashboard 配）

新订阅默认的评论触发范围，可在 **Dashboard → 机器人默认设置** 里按机器人配置：

| 取值 | 含义 |
|------|------|
| 仅 `@` 我的评论（默认） | 只有评论里 `@机器人` 才触发，防刷屏 |
| 所有新评论 | 该文档任何新评论都触发，适合专用文档 |

对应 `bots.json` 字段 `docSubscribeDefaultMode`（`"all"` 开启「所有新评论」；缺省为「仅 @」）。

## 授权

订阅文档评论需要**「文档权限」的用户授权**（读写评论 + 订阅事件），与通用 [`/login`](/slash-commands) 的权限不同。第一次 `/subscribe-lark-doc` 时机器人会直接给你一条**带文档权限的授权链接**，点开授权后重新发命令即可。

此外需要在**飞书开放平台后台**为该应用做两件事（一次性）：

1. 「权限管理」里开通文档评论相关权限（`docs:document.comment:read` / `docs:document.comment:create` / `docs:document.subscription` 等）并发布版本；
2. 「事件订阅」里添加 **`drive.notice.comment_add_v1`（云文档新增评论）** 事件，订阅方式用长连接。

缺权限或没订阅事件时，机器人在启动自检里会私信管理员提示。

## 生命周期

- `/close` 关闭会话时，自动退订其绑定的所有文档。
- daemon 重启后自动恢复仍活跃会话的文档订阅。

## 限制 & 说明

- **必须逐文档订阅**：飞书云文档事件是 per-instance 的，没有「订一次收全部文档」的能力（用户身份也只能订阅自己拥有 / 管理的文档，仍需逐篇订）。
- **嵌套回复的兜底**：极少数评论（如某些已解决 / 受限的评论）飞书不允许 API 回复，此时机器人会自动改为**新建一条全文评论**作为答复，保证回复总能落到文档评论区。
- 文档评论是纯文本通道，富交互（卡片 / 按钮 / 终端链接）仍走飞书话题。
